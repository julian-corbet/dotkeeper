dotkeeper

Your repos, everywhere. Automatically.

Real-time P2P file sync between your machines via embedded Syncthing. Scheduled git backup for versioned history. One binary. No servers. No accounts.

Install See it in action

The problem

Working across machines is fragile

You edit code on your laptop, push to GitHub, pull on your desktop. But what about uncommitted work? Config files? Dotfiles that aren't in a repo? You either remember to push everything or you lose it.

dotkeeper fixes this

Every file change syncs instantly between your machines via Syncthing. Git auto-commits and pushes on a schedule so nothing is ever lost. You don't think about it — it just works.

Why not just use...

Git alone?

You have to remember to commit and push. Uncommitted work doesn't sync. Binary files bloat the repo. Merge conflicts require manual intervention.

dotkeeper syncs all files instantly via Syncthing (including uncommitted work), then auto-commits to git on a schedule. Best of both worlds.

Syncthing alone?

No version history. If a file gets corrupted or accidentally deleted, it's gone everywhere. No way to see what changed or roll back.

dotkeeper adds git on top of Syncthing. Every change is versioned. Roll back to any point. Push to your existing git remotes for backup.

Cloud sync (Dropbox, iCloud)?

Your code lives on someone else's server. File locking breaks dev tools. No git integration. Costs money. Not available on BSDs or servers.

dotkeeper is peer-to-peer. Your files never touch a third-party server. Works everywhere Go compiles: Linux, macOS, Windows, BSDs.

Three commands to sync everything

1

Initialize your first machine

dotkeeper init

Generates a Syncthing identity, creates your config, starts the embedded sync engine.

2

Join from other machines

dotkeeper join XXXXXXX-XXXXXXX-...

Pass the device ID shown by init. Config syncs automatically. Machines discover each other via Syncthing's global network.

3

Add your repos

dotkeeper add ~/dotfiles ~/projects/myapp

Files propagate in real-time between machines. Git backs up on your schedule — hourly, daily, weekly.

Built right

Single binary, zero deps

Syncthing is compiled in. Download one file, run it. No runtime dependencies except git. No daemon to install separately.

Isolated by default

Runs its own Syncthing instance on dedicated ports (18384, 12000, 11027). Never touches your existing Syncthing installation or config.

Conflict-free git

Each machine gets a timer slot. Machine A pushes at 02:00, machine B at 02:05. Staggered by design so pushes never collide.

Every platform

Linux, macOS, Windows, FreeBSD, OpenBSD, NetBSD. Native service management: systemd, launchd, Task Scheduler, cron.

Privacy-first

No telemetry, no crash reporting, no accounts, no cloud. Syncthing's full P2P network stack with all reporting disabled.

Smart ignore patterns

Auto-generated .stignore skips build artifacts, lockfiles, editor state, SQLite journals, OS metadata. Syncs what matters.

Architecture

Machine A
dotkeeper binary
Embedded Syncthing
Git timer (slot 0)
P2P sync (real-time) Git remote (scheduled)
Machine B
dotkeeper binary
Embedded Syncthing
Git timer (slot 1)

Three config files

machine.toml — local identity (name, slot). Never leaves the machine. config.toml — repos, machines, ignore patterns. Synced via Syncthing. dotkeeper.toml — per-repo breadcrumb log. Tracked in git.

Dual sync strategy

Syncthing provides instant, bidirectional file sync for the working state. Git provides versioned history, remote backup, and rollback. Together they cover the full lifecycle.
